5. Windows 작업 스케줄러 등록 방법
이제 수정된 배치 파일들을 Windows 작업 스케줄러에 등록하는 방법을 설명합니다.

A. trading_start.bat 등록 (매일 1회 실행)
이전 답변에서 설명드린 것과 동일하게 등록하되, 
main.py 종료 로직이 이제 trading.py 내부에 있으므로, 동작 탭에서 taskkill 관련 설정은 필요 없습니다.

작업 스케줄러 열기: Windows 검색 창에 "작업 스케줄러" 또는 "Task Scheduler" 입력 후 실행.
새 작업 만들기: 오른쪽 작업 창에서 작업 만들기... 클릭.
일반 탭:
    이름: 자동매매_일일_시작_종료
    설명: 자동매매 시스템의 일일 시작 및 종료 작업을 관리합니다.
    사용자 계정: 시스템을 실행할 계정 선택. 가장 높은 권한으로 실행 체크.
    구성: Windows 10 또는 해당 Windows 버전 선택.

트리거 탭:
    새로 만들기... 클릭.
    작업 시작: 예약 시
    설정: 매일 선택, 1일마다 반복.
    시작: 현재 날짜와 시간을 **08:30:00**으로 설정.
    고급 설정: 사용 체크.

확인 클릭.

동작 탭:
    새로 만들기... 클릭.
    동작: 프로그램 시작
    프로그램/스크립트: trading_start.bat 파일의 전체 경로 입력 (예: C:\your_project_root\trading_start.bat)
    시작 위치: trading_start.bat 파일이 있는 디렉토리 경로 입력 (예: C:\your_project_root)

확인 클릭.

조건 탭 (권장):
    컴퓨터 유휴 상태일 때만 시작: 체크 해제.
    AC 전원에 있을 때만 시작: 노트북이라면 체크 해제 (AC 전원 연결 권장).
    네트워크 연결 사용 가능: 어떤 연결이든 선택.

설정 탭:
    요청 시 작업 실행 허용: 체크.
    예정된 시작이 누락된 경우 즉시 작업 실행: 체크.
    실행 중인 작업이 멈추면 강제로 중지: 8시간 (08:30 ~ 15:40까지 충분한 시간 부여).
    기존 인스턴스가 실행되고 있으면: 새 인스턴스 시작 안 함.

최종 확인: 모든 설정 검토 후 확인 클릭.

B. trading_monitor.bat 등록 (장중 10분마다 실행)
이 작업은 장이 열리는 시간(09:00 ~ 15:30) 동안 
10분 간격으로 main.py가 정상적으로 실행 중인지 확인하고, 
그렇지 않으면 재시작하는 역할을 합니다.

작업 스케줄러 열기: (위와 동일)

새 작업 만들기: 오른쪽 작업 창에서 작업 만들기... 클릭.

일반 탭:
    이름: 자동매매_장중_모니터링
    설명: 자동매매 시스템(main.py)의 장중 실행 상태를 10분마다 모니터링하고 비정상 종료 시 재시작합니다.
    사용자 계정: 시스템을 실행할 계정 선택. 가장 높은 권한으로 실행 체크.
    구성: Windows 10 또는 해당 Windows 버전 선택.

트리거 탭:
    새로 만들기... 클릭.
    작업 시작: 예약 시
    설정: 매일 선택, 1일마다 반복.
    시작: 현재 날짜와 시간을 **09:00:00**으로 설정.

고급 설정:
    반복 작업(Repeat task every): 10분 선택.
    기간(for a duration of): 6시간 30분 (09:00부터 15:30까지).
    사용 체크.

확인 클릭.

동작 탭:
새로 만들기... 클릭.

동작: 프로그램 시작
    프로그램/스크립트: trading_monitor.bat 파일의 전체 경로 입력 (예: C:\your_project_root\trading_monitor.bat)
    시작 위치: trading_monitor.bat 파일이 있는 디렉토리 경로 입력 (예: C:\your_project_root)

확인 클릭.

조건 탭 (권장):
    컴퓨터 유휴 상태일 때만 시작: 체크 해제.
    AC 전원에 있을 때만 시작: 노트북이라면 체크 해제 (AC 전원 연결 권장).
    네트워크 연결 사용 가능: 어떤 연결이든 선택.

설정 탭:
    요청 시 작업 실행 허용: 체크.
    예정된 시작이 누락된 경우 즉시 작업 실행: 체크.
    실행 중인 작업이 멈추면 강제로 중지: 15분 (모니터링 주기보다 약간 길게).
    기존 인스턴스가 실행되고 있으면: 새 인스턴스 시작 안 함.

최종 확인: 모든 설정 검토 후 확인 클릭.

최종 확인 사항:

config/settings.py의 모든 YOUR_... 값을 실제 환경에 맞게 정확히 설정했는지 다시 한번 확인하세요. 
특히 크레온 로그인 정보와 텔레그램 토큰/채팅 ID는 필수입니다.

CONDA_PATH와 CONDA_ENV_NAME 변수가 
사용자 환경의 아나콘다 설치 경로 및 가상 환경 이름과 정확히 일치하는지 확인하세요.

PROJECT_ROOT 변수가 프로젝트의 실제 루트 디렉토리 경로와 일치하는지 확인하세요.

Creon Plus HTS가 설치되어 있고, 작업 스케줄러가 실행되는 사용자 계정으로 로그인되어 있어야 합니다.

로그 파일(logs 디렉토리 내)을 주기적으로 확인하여 시스템이 예상대로 동작하는지, 오류가 발생하는지 모니터링하세요.

이러한 설정들을 통해 자동매매 시스템이 매일 자동으로 시작, 운영, 모니터링 및 종료될 수 있습니다.
